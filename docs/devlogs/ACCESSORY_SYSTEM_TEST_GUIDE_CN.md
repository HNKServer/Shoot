# 国服饰品系统真机测试指南

本指南用于验证 NPPS4 的完整饰品生命周期：读取、制作、收藏、装备、强化、重制、出售、持久化，以及演唱会数值生效。

## 测试前准备

1. 备份服务端 `data/main.sqlite3`。
2. 准备至少 8～12 张不收藏、不是主界面伙伴、也没有装备饰品的普通社员；制作饰品会永久消耗所选社员。
3. 准备足够金币。
4. 每执行一个操作后都退出对应页面再重新进入；关键步骤还应彻底重启游戏和服务端，以验证持久化。
5. 出错时生成 Wrapper 诊断报告，并截取从操作前 5 秒到错误后 10 秒的 Logcat。重点搜索接口名：
   `accessoryAll`、`accessoryMaterialAll`、`accessoryTab`、`createAccessory`、`favoriteAccessory`、`wearAccessory`、`mergeAccessory`、`saleAccessory`。


## 可选：临时获得强化/重制零件

正常制作饰品不需要预先拥有零件；只要消耗社员即可生成饰品。若要专门测试 Glass Part/Jewel Part，可临时编辑 `external/login_bonus.py`，把 `get_rewards` 的返回值改成一项饰品奖励：

```python
async def get_rewards(day: int, month: int, year: int, context):
    # add_type=1002 表示饰品/饰品材料。下面示例发放 20 个 N Glass Part。
    return (1002, 1008, 20, None)
```

重启服务端后，在尚未领取当天登录奖励的测试账号上登录即可领取。测试 Jewel Part 时把 `1008` 改为 `1013`；测试更高稀有度时使用 1004～1007 或 1009～1012。完成测试后恢复原来的 `login_bonus.py`。

注意：同一账号同一天只会领取一次登录奖励。最稳妥的做法是使用一个专门的测试账号，并在修改奖励前备份 `data/main.sqlite3`。

## A. 页面与初始状态

进入“社员/成员”相关菜单中的饰品页面。

预期：
- 页面正常打开，不闪退。
- μ's、Aqours 等分页可切换。
- 空账户会显示空饰品列表，而不是通信错误。
- 日志中 `unit/accessoryAll`、`unit/accessoryMaterialAll`、`unit/accessoryTab` 均返回 200。

## B. 饰品制作

1. 进入“制作饰品”。
2. 先选择一张低稀有度、未收藏、非伙伴社员进行制作。
3. 再用多张不同等级/技能等级的社员制作一次。
4. 记录制作前后的金币、社员数量和新饰品。

预期：
- 被选择的社员消失。
- 金币按社员强化/合成成本扣除。
- 新饰品出现在饰品列表。
- 同一张请求不会生成两个饰品。
- 若所选单张社员存在专属饰品映射，应得到专属饰品；否则按制作价值进入饰品抽选。
- 收藏社员、主界面伙伴、或正装备饰品的社员不能被消耗。

## C. 收藏标记

1. 对新饰品点亮收藏/锁定。
2. 退出页面后重新进入，并重启游戏确认标记仍在。
3. 尝试把该饰品用于强化材料或出售。
4. 取消收藏后再尝试。

预期：
- 收藏状态持久保存。
- 收藏中的饰品不能被强化消耗、重制消耗或出售。
- 取消收藏后才能消耗。

## D. 装备、替换与卸下

1. 把饰品装备给一个处于启用状态的社员。
2. 退出页面并重进，确认装备关系仍在。
3. 把同一饰品改装给另一名社员。
4. 给原社员换另一件饰品。
5. 卸下饰品。

预期：
- 一名社员最多装备一件饰品；一件饰品最多属于一名社员。
- 改装时旧关系自动解除，不产生重复装备记录。
- 等待室/未启用社员不能装备。
- 已装备饰品不能被出售或作为强化/重制材料。
- 重启服务端后装备关系仍然存在。

## E. 普通强化

普通强化使用其他未收藏、未装备的饰品，或 Glass Part（强化零件）。国服主数据中的强化材料为：

- 1004：U Glass Part
- 1005：SS Glass Part
- 1006：S Glass Part
- 1007：R Glass Part
- 1008：N Glass Part

步骤：
1. 选一件低等级饰品作为本体。
2. 使用一件普通饰品进行强化，记录 EXP、等级、金币。
3. 再用 Glass Part 强化。
4. 一次投入多件材料，使饰品达到当前等级上限。

预期：
- 材料饰品/零件数量减少。
- 本体 EXP 与等级提高，金币正确扣除。
- 达到当前上限后显示满级，额外 EXP 不应丢失；完成重制提高上限后，保留的溢出 EXP 应继续参与升级。
- 不能把本体自身、收藏饰品、已装备饰品当作材料。

## F. 重制/突破上限

重制使用同 ID 的重复饰品，或 Jewel Part（重制零件）：

- 1009：U Jewel Part
- 1010：SS Jewel Part
- 1011：S Jewel Part
- 1012：R Jewel Part
- 1013：N Jewel Part

步骤：
1. 把本体练到当前上限。
2. 使用一件完全相同 ID 的重复饰品执行重制。
3. 再用符合稀有度要求的 Jewel Part 重制。
4. 重复直到达到最大重制次数。

预期：
- `rank_up_count` 每次按实际材料增加。
- 最大等级随重制次数提高，最多 4 次，并且不超过主数据绝对上限。
- 重复制品必须与本体 `accessory_id` 相同。
- Glass Part 不能用于重制；Jewel Part 不能用于普通强化。
- 超过剩余重制次数的伪造/过量请求必须被拒绝，不能吞掉多余材料。

## G. 出售

1. 出售一件未收藏、未装备的饰品。
2. 出售少量 Glass/Jewel Part。
3. 尝试出售收藏或已装备饰品。

预期：
- 饰品或材料数量减少，金币按当前等级主数据增加。
- 收藏或已装备饰品不能出售。
- 达到金币上限时，奖励箱标志和礼物箱行为应符合客户端显示，不应丢失奖励。

## H. 演唱会数值生效

1. 记录某支队伍未装备饰品时的 Smile/Pure/Cool 总值。
2. 给队内社员装备属性明显的饰品。
3. 重新打开队伍详情并开始一场演唱会。
4. 比较队伍总属性和得分表现。

预期：
- 装备饰品后，对应 Smile/Pure/Cool 属性进入队伍计算。
- 卸下后数值恢复。
- 不能只在 UI 中显示装备，却完全不影响服务端 live deck 计算。

## I. 持久化与异常回归

完成制作、收藏、装备、强化、重制后：

1. 强制结束游戏并重启。
2. 重启 NPPS4 服务端。
3. 再次进入饰品页和队伍页。

所有饰品、EXP、等级、重制次数、收藏状态、材料数量和装备关系都应保持一致。

# NPPS4 Android Wrapper v2.7

## Master DB auto extraction

NPPS4 has two different data classes:

- Runtime/server state DB: generated by NPPS4, contains accounts and progress.
- Read-only master DBs: game metadata such as units, lives, items and scenarios.

Earlier wrapper builds incorrectly required users to manually place the master DBs under `db_root`.
This version adds automatic extraction from the selected `list_CN_Android` archive folder.

The wrapper now scans the CN archive ZIP files for these files:

- `achievement.db_`
- `effort.db_`
- `exchange.db_`
- `game_mater.db_`
- `item.db_`
- `live.db_`
- `museum.db_`
- `scenario.db_`
- `subscenario.db_`
- `unit.db_`

Extracted DBs are written to the app workspace:

`Android/data/moe.honoka.npps4wrapper/files/npps4/data/db_auto/`

This keeps the large CDN archives in the public folder and only stores the small extracted master DBs in the app workspace.

## UI

A new button has been added under path mapping:

`从 CN 数据包自动提取 master DB`

Service startup also attempts this extraction automatically when `db_root` is missing or incomplete.
